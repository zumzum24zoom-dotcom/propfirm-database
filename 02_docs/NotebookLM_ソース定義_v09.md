## 📋 このソースの役割

このドキュメントは、Prop Firmの情報をNotebookLMで出力する際の**構成要素の定義・品質基準・出力形式・用語統一ルール**を定義します。

- **分析項目リストを冒頭に明示**（何を集めるかの全体像）
- **ページ構造別に出力形式を定義**（DBP_01 / DBP_02 / 攻略記事）
- **全テーブルに見出し・説明文を必須化**（文脈を明確にする）
- **用語統一・計算ロジック・判定基準を厳格化**（品質の一貫性を確保）

このソースに「望む情報の形（構成要素・表記・判定・計算）」を固定しておくことで、プロンプトは「○○の構成要素をください」と呼び出すだけで足りる。

## 📤 出力形式の基本ルール

### 各テーブルの必須要素

すべてのテーブルは以下の構成で出力すること:

1. **見出し** — Markdown見出し記法（`##` または `###`）
2. **説明文** — テーブルの目的・構成を1〜2文で記載
3. **Markdownテーブル本体** — 純粋なMarkdown table記法（`| 列 | 列 |`）
4. **補足注記**（必要に応じて）— 計算ロジック・判定基準・特記事項

### 禁止事項

- ❌ **HTMLタグ（原則禁止）**: `<table>`、`<div>` 等のHTMLタグは絶対使用禁止
  - **例外**: Markdownテーブル内のセル改行に限り `<br>` タグは使用可能
- ❌ **過剰な挨拶**: 冗長な前置き・AI自己紹介・社交辞令は不要
- ❌ **コードブロック**: Markdownテーブルを ` ```markdown ` で囲まない

### 推奨事項

- ✅ セクション区切りには水平線（`---`）を使用可能
- ✅ 各テーブルの文脈を明確にする簡潔な説明は必須
- ✅ 数値・データの正確性を最優先

### 口調（厳守）

- 説明文・詳細列・補足注記はすべて**常体（だ・である調・体言止め）**で記述する。
- **「です・ます」調は禁止**。「〜が必要です」→「〜が必要」、「許可されています」→「許可されている」、「〜となります」→「〜となる」。
- 客観的事実の記述に徹し、丁寧語・敬体を混在させない。

### テーブル7: ルール早見表

**見出し**: `## ルール早見表`

**説明文**: 該当プランの項目（Steps + P01〜P19）をC/F/差分/詳細の5列で出力。詳細列を含む形式がPage Makerの取込フォーマット（rd_*スロット）と一致する。

**構成**:

| 項目 | C | F | 差分 | 詳細 |
| :--- | :--- | :--- | :--- | :--- |
| Steps | 2 | ー | ー | 2-Step構成。Challenge 1次・2次を通過してFundedへ移行。 |
| P01: 利益目標 | 10% | ー | ー | Challenge・Funded共に利益目標10%。Funded最低出金額の設定なし。 |
| P02: 最低取引日数（合格） | 3日 | ー | ー | Challenge・Funded共に最低3日の取引が必要。 |
| P03: 日次損失 | 5% | 3% | TRAP | Challengeでは5%の日次損失制限。Fundedでは3%に引き下げられる。 |
| P04: 日次損失タイプ | Trailing（Balance EOD） | Trailing（Balance EOD） | ー | Challenge・Funded共にTrailing（Balance EOD）。日次終値残高を基準にTrailingする。 |
| ... | ... | ... | ... | ... |
| P18: 最大ポジション制約 | 無 | 無 | ー | Challenge・Funded共に最大ポジション数について明示的に制限する記載はない。 |

※ **P19（禁止行為）はこのルール表に含めない**（別シェイプ）。ルール表の直後に、下記の独立した禁止行為テーブルを出力する。

**禁止行為テーブル（各プランのルール表の直後に出力）**:

| 行為名 | 備考 | 行為 | 処分 |
| :--- | :--- | :--- | :--- |
| Gap Trading | | 価格ギャップを狙った取引 | 即時強制閉鎖 |
| HFT | | 高頻度自動取引 | 即時強制閉鎖 |
| レイテンシーアービトラージ | | 価格遅延差を突く取引 | 即時強制閉鎖 |

**列の定義**:
- **C**: Challenge値
- **F**: Funded値。C/Fが同一またはF非該当 → 「ー」
- **差分**: C/Fが同値またはF=ー → 「ー」 / Funded側が緩い → 「EASE」 / Funded側が厳しい → 「TRAP」
- **詳細**: 【詳細がルールの正本】詳細を読めばそのルールについて不明な点がない状態で記述すること。補足・おまけ扱い禁止。C/F値の数字を詳細に再掲して、条件・例外・ペナルティを含む完結した本文として書くこと。条件付き許可（※有）の場合は条件の中身を含めルール全容を記述。**常体（だ・である調・体言止め）で書き、「です・ます」調は禁止**

**同値の扱い（厳守）**:
- C/Fが同じ値のときも、**省略せず両列に同じ値を素直に記載**する。差分列は「ー」。
- 「ー」は **その項目が構造上そもそも存在しない場合のみ**使う（例: Funded側に合格目標がない）。「Challengeと同じ」を意味する省略記号としては使わない。

**差分判定例**:
- 利益目標: C=10%（Challenge合格目標）/ F=2%（Funded最低出金額）→ F側が緩い → EASE
- 日次損失: C=5%, F=3% → F側が厳しい（少ない損失で違反）→ TRAP
- 時間制限: C=30日, F=無制限 → F側が緩い → EASE

**厳守ルール（用語統一）**:
- フェーズ名称は必ず「Challenge」「Funded」を使用
- 公式サイトで「Evaluation」「Assessment」「Phase 1/2」「Master」「Live」等の表記があっても、出力時は「Challenge」「Funded」に統一変換すること

**厳守ルール（数値・単位）**:
- すべての数値に単位を必ず付与（%、$、日、分、ロット等）
- 単位なしの数値（例: 「3」「5」）は絶対に禁止

**厳守ルール（詳細列の書き出し）**:
- 差分=ー（C/F同値またはF非該当）→ 書き出しは「Challenge・Funded共に～」
- 差分=EASE / TRAP（C/Fが異なる）→ 書き出しは「Challengeでは～。Fundedでは～」
- 値が「無」の項目 → 後述「『無』の項目の詳細定型文」に従う
- P19（禁止行為）はこのルール対象外（別シェイプの独立テーブル。後述「P19（禁止行為）の記載方法」を参照）

**「無」の項目の詳細定型文（厳守）**:
そのルールで縛られない（＝「無」）項目の詳細列は、項目名を入れた定型文で統一する。素っ気ない「記載なし」とは書かない。

- C・F共に「無」（差分=ー）→「Challenge・Funded共に〔ルール名〕について明示的に制限する記載はない」
- Challengeのみ「無」→「Challengeでは、〔ルール名〕について明示的に制限する記載はない」
- Fundedのみ「無」→「Fundedでは、〔ルール名〕について明示的に制限する記載はない」

例（P10 ニュース取引制限がC・F共に無）:「Challenge・Funded共にニュース取引について明示的に制限する記載はない」

**最低取引日数（P02/P02b）の詳細の書き方（特例・厳守）**:
最低取引日数は **C=合格要件 / F=出金要件** であることが列で自明なので、詳細文に「（合格）」「（出金）」の括弧書きは**付けない**。ルール名は「最低取引日数」のみを使い、C側・F側それぞれを素直に1文で書く。
- C側に記載なし → 「Challengeでは最低取引日数について明示的に制限する記載はない。Fundedでは〔出金に必要な最低取引日数〕」
- F側に記載なし → 「〔合格に必要な最低取引日数〕。Fundedでは最低取引日数について明示的に制限する記載はない」
- C・F共に記載なし → 「Challenge・Funded共に最低取引日数について明示的に制限する記載はない」
- 両方に日数指定あり → 「Challengeでは合格に〔C日〕の取引が必要。Fundedでは出金に〔F日〕の取引が必要」

**C/Fの型ルール**:
- 利益目標のみ特例: C=Challenge合格目標(%) / F=Funded最低出金額（初期残高に対する%または固定金額。構造上設定がない場合はー）
  - 複数ステップで合格目標が分かれる場合、C列は **「X% / Y% / Z%」のスラッシュ区切り**で記す（例: 2-Step→「8% / 5%」）。「1次/2次/3次」のラベルは**使わない**
  - 詳細も同様に「Challengeの合格目標は X% / Y% / Z%」のように書き、「1次・2次」等のステップ番号ラベルを付けない（各ステップが同値でもスラッシュ区切りで素直に並べる）
- 数字型（最低取引日数・日次損失%・最大損失%）= 数値+単位。制限が存在しない場合は「無」、項目が構造上存在しない場合は「ー」
- Lossタイプ（日次損失タイプ・最大損失タイプ）= DD計算基準6分類のみ
- その他の制約 = 有 / ※有 / 無 / ー（有=制限あり / ※有=条件付き許可・条件の中身を詳細列に / 無=そのルールで縛られない / ー=項目が構造上存在しない）

**P19（禁止行為）の記載方法（別シェイプ・独立テーブル）**:

禁止行為は早見表（項目|C|F|差分|詳細）とは構造が異なるため、ルール表には**含めない**。各プランのルール表の直後に、**独立した禁止行為テーブル**を出力する。

```
| 行為名 | 備考 | 行為 | 処分 |
|--------|------|------|------|
| HFT | | 高頻度自動取引による市場の非効率性の搾取 | 即時失格 |
| レイテンシーアービトラージ | | 遅延市場データを利用した裁定取引 | 即時失格 |
```

- **行為名**: ファームが明示的に禁止と名指しした行為の名称（1行=1行為・省略禁止）
- **備考**: 極稀に C/F で可否が異なる場合のみ記載（例:「Cは許可/Fは禁止」）。通常は**空欄**
- **行為**: その行為の具体的な説明（何をする行為か）
- **処分**: 違反時のペナルティ（例: 即時失格・利益没収・アカウント停止）
- 禁止行為が無いプランは、このテーブル自体を省略する
- ファームが明示的に禁止と名指しした行為のみを記載（後述の「禁止行為の定義と境界」を厳守）
- C/Fで禁止行為が分かれることは原則想定しない（分かれる稀なケースのみ備考で表現）

## 🚫 禁止行為（P19）の定義と境界（厳守）

禁止行為まわりの誤分類を防ぐため、定義と境界を明確に定める。

### 定義

**禁止行為（P19）とは、ファームが公式に「やってはならない」と明示的に名指しした行為のことだけを指す。**

- 判定の物差しは唯一「**ファームが明示的に禁止と名指ししたか**」
- 名指しされた行為（例: HFT、ティックスキャルピング、レイテンシーアービトラージ、Gap Trading、両建て相殺、マーチンゲール等）→ P19に**行為名をそのまま列挙**
- 名指しされていない制約・条件 → P19には入れない

### 境界ルール（取引ルールとの分離）

**P19（禁止行為）と、取引ルール項目（P10〜P18）を混同しない。**

- **取引ルール（P10〜P18）** = そのカテゴリの制約・条件の有無を扱う。例: P15「スキャルピング制約」は、スキャルピングが許可されるか・ロットや保有時間等の条件は何か、を扱う
- **禁止行為（P19）** = ファームが明示的に禁止と名指しした行為名の列挙

### 厳守事項

- ❌ **P19に列挙される行為名を、取引ルール項目（P10〜P18）に流用・当てはめてはならない**
  - 例: 「ティックスキャルピング」「HFT」はファームが明示的に禁止と名指しした行為なので**P19に入る**。これらをP15「スキャルピング制約」に書いてはならない。P15はスキャルピングという手法に対する制約・条件のみを扱う
- ❌ ファームが明示的に禁止と名指ししていないものを、P19（禁止行為）に入れてはならない

## 🔧 計算ロジック（厳守事項）

### 最短合格日数の計算

各Stepの `MAX(最低取引日数（合格）, 一貫性達成最速日数)` の合計。

**詳細**:
- 一貫性はBest Day系(X%)のみ `ceil(100÷X%)` で算出
- 時間制約系・Loss系は除外
- 「条件として存在しない」項目はinputから除外（0で埋めない）
- 全条件が不存在の場合 → フォールバック: `1日 × Step数`
- Instant Funding = `0日`
- 「0日」「制限なし」「即日理論値」は禁止。全条件不存在でも最低`1日×Step数`

### 最速出金日数の計算

1. `X = MAX(最低取引日数（出金）, 一貫性(Funded側))` を算出
2. 全不存在 → `X=0`
3. 出金頻度に応じて:
   - 随時 → `X`
   - 毎週 → `MAX(7, X)`
   - 隔週 → `MAX(14, X)`
   - 月次 → `MAX(30, X)`
4. 初回限定特例あれば → 最速出金(初回) = `X`

**最低利益は律速計算の対象外**（金額/割合は市場条件依存で日数に変換できないため）。

### トータル日数

`トータル日数 = 最短合格 + 最速出金`

### 不在表記のルール（早見表C/Fセルの値）

制限ルールは本来「有るか／無いか」の二択である。出所が公式明記か沈黙かは「縛られない」という結論を変えないため、両者を「無」に一本化する。

| 状況 | 表記 |
| :--- | :--- |
| 制限あり | 「有」 |
| 条件付きで許可（内容に注意） | 「※有」 |
| そのルールで縛られない（公式が「なし」と明記／公式に制限の記載が見つからない、の両方を含む） | 「無」 |
| 該当項目自体が構造上適用されない | 「ー」（全角ハイフン） |

**区別**:
- 「無」=制限ルールが存在しない（出所が明記か沈黙かを問わない）。
- 「ー」=項目自体が構造上そもそも存在しない（例: Funded側に合格目標がない）。「無」とは別物。
- **旧表記「なし（明示）」「記載なし」は廃止**。これらはすべて「無」に統一する。
- 絵文字は付けない。

**補足**: セル表示は「無」に統一するが、詳細列の文言では出所の事実を「〔ルール名〕について明示的に制限する記載はない」と表現し、断定の「制限なし」とは書かない（公式更新・規約PDF奥への追記など後の追跡可能性を残すため）。
