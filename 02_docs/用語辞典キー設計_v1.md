# 用語辞典キー設計 v1（確定版）

策定日: 2026-05-27
位置づけ: 用語辞典を「単語の正本」とするためのキー定義
状態: 設計確定（実装未着手）

---

## 0. 設計思想（マスター定義）

**用語辞典＝国語辞典である。**

- 結果（語句）だけを載せる。人間も計算機も、結果で理解・分類するから。
- 定義に収まる語句を分類する。
- 計算式（手続き）は載せない。見えないところ（ソース定義）に置く。

この思想により、用語辞典の責務は「語句の分類と正規表記の確定」に純化される。
算出式・計算手続きは越境させない。

### 正本の二層構造

| 層 | 正本 | 役割 |
|---|---|---|
| 単語 | 用語辞典 | 語句・正規表記の唯一の典拠（本ドキュメント） |
| 記事 | 3作成ボタンが参照する「文章の構造定義」 | 構造（骨格）＋ 語句スロット（用語辞典参照）で組み立て |

ボタンが生成する文字列は正本ではない。生成の入力（用語辞典＋構造定義）が正本。

## 1. スキーマ拡張案

現状の用語辞典スキーマ `{term, category, definition, explanation, trivia}` に
参照層を追加する。既存の「読み物」機能は温存。

```
{
  key:        "P02",                  // 参照ID（一意・不変）
  term:       "最低取引日数（合格）",   // 正規表記（=正本の値）
  type:       "label",                // label | notation
  category:   "Plan項目",
  definition: "...",                  // 既存（読み物・任意）
  explanation:"...",                  // 既存（任意）
  trivia:     "..."                   // 既存（任意）
}
```

- `key`: 他箇所から `{glossary:key}` で引く参照ID。一意・不変。
- `term`: 正規表記。これが「単語の正本」の値。
- `type`: `label`（項目ラベル・計算結果列名）/ `notation`（正規表記）の2種。
  ※ 国語辞典思想により `calc`（算出式）typeは設けない。
