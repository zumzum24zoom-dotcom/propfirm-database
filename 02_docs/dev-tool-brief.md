# 専用開発ツール — プロジェクトブリーフ

## 背景・課題

Prop Firm Challengers（Hugo サイト）の開発で、毎回 UI の仕様を一から説明・実装するコストが高い。  
具体的には：
- 「カード型リスト with ドラッグ並べ替え・ブラシ選択・セクション管理」
- 「スロット入力パネル with ロック・アーカイブ・AI アシスト」
- 「ヘッダー固定パネル with 編集ターゲット表示」

…といった複合 UI を毎ツール再実装している。

**解決したいこと：** よく使う UI パターンを「セット」として持ち、  
新しいツール開発時はパラメータを渡すだけで組み立てられるようにする。

---

## コンセプト

> **UI コンポーネントライブラリ + ツール生成環境**  
> ゼロから説明せずに意図が伝わるレベルの共通言語を持つ開発基盤

名称案：`DevKit` / `PFD DevKit`

---

## 想定用途

| 用途 | 例 |
|------|----|

| データ管理 UI | Firm / Plan / Glossary 管理画面 |
| 分析・可視化ツール | ランキング・チャート系 |

## UI 基本セット（必須コンポーネント）

### リスト系
- **カードリスト** — ドラッグ並べ替え、ブラシ複数選択、セクション折りたたみ、完了管理（上下 🔺🔻）
- **フラットリスト** — チェックボックス or ラジオ選択、一括操作バー

### パネル系
- **スロットパネル** — ラベル・入力・ロック・アーカイブ・AI アシスト付き
- **編集ターゲットパネル** — 現在作業中のアイテム表示、ヘッダー固定
- **プレビューパネル** — Markdown / HTML レンダリング

### 操作系
- **コピーボタン** — クリックで clipboad コピー、完了フィードバック付き
- **ドロップダウンメニュー** — `▾` トリガー、アイテムクリックで閉じる
- **トースト通知** — 2.5 秒自動消滅
- **セクション追加フォーム** — インライン入力 + Enter 確定

## 技術スタック

| 項目 | 選定 |
|------|------|
| ランタイム | ブラウザ単体（CDN React + Babel） |
| フレームワーク | React 18（JSX in `<script type="text/babel">`） |
| スタイル | インライン style（CSS 変数なし、定数 `C.*`） |
| 保存 | localStorage + JSON ファイル書き出し |
| 配置 | `01_tools/` 以下の単一 HTML ファイル |

## 参照実装

`01_tools/page-maker-v11.html` が現時点の最も完成度の高い実装。  
以下が流用可能：

- `VerticalTabs` — カードリスト + セクション + ブラシ選択の全実装
- `Btn`, `CopyBtn` — 共通ボタン
- `useResize` — ペインリサイズ hook
- デザイントークン定数 `C`、フォント定数 `font`
- `callAPI` — Cloudflare Worker プロキシ経由 Claude 呼び出し

## 開発方針

1. **コンポーネント単位で独立** — 各 UI セットは props だけで動作する純粋コンポーネント
2. **単一 HTML 原則** — CDN + Babel でブラウザ完結、Node 不要
3. **デザイントークン共通** — 全ツールで `C.*` を使い回す
4. **マスター主導** — 戦略・構成決定はマスター。ソートーは技術実装に徹する

---

## 最初のアクション候補

a. `devkit-v1.html` としてコンポーネントカタログページを作成（ショーケース）  
b. 既存コンポーネントを関数単位で抽出・整理してライブラリ化  
c. 新ツール（別途指定）にコンポーネントセットとして組み込み

→ **マスターが方向を決定してから着手**
