## 📋 このソースの役割

このドキュメントは、Prop Firmの情報をNotebookLMで出力する際の**構成要素の定義・品質基準・出力形式・用語統一ルール**を定義します。

- **分析項目リストを冒頭に明示**（何を集めるかの全体像）
- **ページ構造別に出力形式を定義**（DBP_01 / DBP_02 / 攻略記事）
- **全テーブルに見出し・説明文を必須化**（文脈を明確にする）
- **用語統一・計算ロジック・判定基準を厳格化**（品質の一貫性を確保）

このソースに「望む情報の形（構成要素・表記・判定・計算）」を固定しておくことで、プロンプトは「○○の構成要素をください」と呼び出すだけで足りる。

## 📤 出力形式の基本ルール

### 各テーブルの必須要素

すべてのテーブルは以下の構成で出力すること:

1. **見出し** — Markdown見出し記法（`##` または `###`）
2. **説明文** — テーブルの目的・構成を1〜2文で記載
3. **Markdownテーブル本体** — 純粋なMarkdown table記法（`| 列 | 列 |`）
4. **補足注記**（必要に応じて）— 計算ロジック・判定基準・特記事項

### 禁止事項

- ❌ **HTMLタグ（原則禁止）**: `<table>`、`<div>` 等のHTMLタグは絶対使用禁止
  - **例外**: Markdownテーブル内のセル改行に限り `<br>` タグは使用可能
- ❌ **過剰な挨拶**: 冗長な前置き・AI自己紹介・社交辞令は不要
- ❌ **コードブロック**: Markdownテーブルを ` ```markdown ` で囲まない

### 推奨事項

- ✅ セクション区切りには水平線（`---`）を使用可能
- ✅ 各テーブルの文脈を明確にする簡潔な説明は必須
- ✅ 数値・データの正確性を最優先

### テーブル7: ルール早見表

**見出し**: `## ルール早見表`

**説明文**: 該当プランの項目（P01〜P19）をChallenge/Funded/差分の3列で比較。

**構成**:

| 項目名 | Challenge | Funded | 差分 |
| :--- | :--- | :--- | :--- |
| P01: 利益目標 | 10% | ー | ー |
| P02: 最低取引日数（合格） | 3日 | ー | ー |
| P03: 日次損失 | 5% | 3% | TRAP |
| P04: 日次損失タイプ | Trailing（Balance EOD） | Trailing（Balance EOD） | ー |
| ... | ... | ... | ... |
| P19: 禁止行為 | Gap Trading, HFT, レイテンシーアービトラージ | 左記に同じ | ー |

**差分列の判定基準**:
- Challenge/Fundedが同値、またはFundedが「ー」 → 差分列は「ー」
- Funded側が**緩い**（トレーダーに有利）→ 「EASE」
- Funded側が**厳しい**（トレーダーに不利）→ 「TRAP」

**判定例**:
- 利益目標: C=10%, F=5% → F側が緩い（低い目標で合格）→ EASE
- 日次損失: C=5%, F=3% → F側が厳しい（少ない損失で違反）→ TRAP
- 制限時間: C=30日, F=無制限 → F側が緩い → EASE

**厳守ルール（用語統一）**:
- フェーズ名称は必ず「Challenge」「Funded」を使用
- 公式サイトで「Evaluation」「Assessment」「Phase 1/2」「Master」「Live」等の表記があっても、出力時は「Challenge」「Funded」に統一変換すること

**厳守ルール（数値・単位）**:
- すべての数値に単位を必ず付与（%、$、日、分、ロット等）
- 単位なしの数値（例: 「3」「5」）は絶対に禁止

**P19（禁止行為）の早見表セル記載方法**:
- セル内に**禁止行為名をそのまま列挙**（カンマ区切り、改行なし）
- **文字数制限なし**。該当する禁止行為名は省略せず全て列挙する
- ファームが明示的に禁止と名指しした行為のみを記載（後述の「禁止行為の定義と境界」を厳守）

## 🚫 禁止行為（P19）の定義と境界（厳守）

禁止行為まわりの誤分類を防ぐため、定義と境界を明確に定める。

### 定義

**禁止行為（P19）とは、ファームが公式に「やってはならない」と明示的に名指しした行為のことだけを指す。**

- 判定の物差しは唯一「**ファームが明示的に禁止と名指ししたか**」
- 名指しされた行為（例: HFT、ティックスキャルピング、レイテンシーアービトラージ、Gap Trading、両建て相殺、マーチンゲール等）→ P19に**行為名をそのまま列挙**
- 名指しされていない制約・条件 → P19には入れない

### 境界ルール（取引ルールとの分離）

**P19（禁止行為）と、取引ルール項目（P10〜P18）を混同しない。**

- **取引ルール（P10〜P18）** = そのカテゴリの制約・条件の有無を扱う。例: P15「スキャルピング制約」は、スキャルピングが許可されるか・ロットや保有時間等の条件は何か、を扱う
- **禁止行為（P19）** = ファームが明示的に禁止と名指しした行為名の列挙

### 厳守事項

- ❌ **P19に列挙される行為名を、取引ルール項目（P10〜P18）に流用・当てはめてはならない**
  - 例: 「ティックスキャルピング」「HFT」はファームが明示的に禁止と名指しした行為なので**P19に入る**。これらをP15「スキャルピング制約」に書いてはならない。P15はスキャルピングという手法に対する制約・条件のみを扱う
- ❌ ファームが明示的に禁止と名指ししていないものを、P19（禁止行為）に入れてはならない

### テーブル9: DD比較テーブル

**見出し**: `## DD比較テーブル`

**説明文**: 全プランの日次損失・最大損失の上限値と計算基準を比較。

**構成**:

| プラン名 | 日次損失上限 | 計算基準（日次） | 最大損失上限 | 計算基準（最大） |
| :--- | :--- | :--- | :--- | :--- |

| LITE 2-Step (Challenge) | 5% | Trailing（Balance EOD） | 10% | Static |
| LITE 2-Step (Funded) | 5% | Trailing（Equity EOD） | 10% | Trailing（Balance HWM） |
| ... | ... | ... | ... | ... |

**フェーズ列のルール**:
- Challenge/FundedでDD値・計算基準が**全て同一**のプラン → 1行（フェーズ列不要）
- Challenge/Fundedで**差分があるプラン**のみ2行に分ける
- Instant Funding → 1行（Fundedのみ）
- 全プランがChallenge/Funded同一 → フェーズ列自体を省略

**補足**:
- 計算基準は6分類のみ使用（後述）
- リセット時刻・Trailing詳細はテーブル外の補足注記に記載

## 🔧 計算ロジック（厳守事項）

### 最短合格日数の計算

各Stepの `MAX(最低取引日数（合格）, 一貫性達成最速日数)` の合計。

**詳細**:
- 一貫性はBest Day系(X%)のみ `ceil(100÷X%)` で算出
- 時間制約系・Loss系は除外
- 「条件として存在しない」項目はinputから除外（0で埋めない）
- 全条件が不存在の場合 → フォールバック: `1日 × Step数`
- Instant Funding = `0日`
- 「0日」「制限なし」「即日理論値」は禁止。全条件不存在でも最低`1日×Step数`

### 最速出金日数の計算

1. `X = MAX(最低取引日数（出金）, 一貫性(Funded側))` を算出
2. 全不存在 → `X=0`
3. 出金頻度に応じて:
   - 随時 → `X`
   - 毎週 → `MAX(7, X)`
   - 隔週 → `MAX(14, X)`
   - 月次 → `MAX(30, X)`
4. 初回限定特例あれば → 最速出金(初回) = `X`

**最低利益は律速計算の対象外**（金額/割合は市場条件依存で日数に変換できないため）。

### トータル日数

`トータル日数 = 最短合格 + 最速出金`

- 合格要件 → **最低取引日数（合格）**

### 単位付与の絶対化

すべての数値に必ず単位（%、$、日、分、ロットなど）を付与する。

**正しい表記例**: ◯「3% / 5%」 ◯「10日 / 14日」 ◯「3分前後 / 5分前後」 ◯「20ロット / 10ロット」
**誤った表記例**: ✕「3」（単位なし） ✕「10 / 14」（単位なし） ✕「3% / 5」（片方だけ単位なし）

### DDタイプ（6分類）

DD計算基準は、推測を交えず以下の6パターンのいずれかから選択すること。括弧は全角（）。

1. **Static**
2. **Trailing（Balance EOD）**
3. **Trailing（Balance HWM）**
4. **Trailing（Equity EOD）**
5. **Trailing（Equity HWM）**
6. **Trailing（Higher B or E EOD）**

**判定根拠キーワード**:

| DDタイプ | 判定根拠キーワード |
| :--- | :--- |
| Static | initial balance / fixed / starting balance |
| Trailing（Balance EOD） | end of day balance / daily reset balance |
| Trailing（Balance HWM） | highest balance / recorded highest balance |
| Trailing（Equity EOD） | end of day equity |
| Trailing（Equity HWM） | highest equity / real-time equity |
| Trailing（Higher B or E EOD） | previous day's balance or equity / higher of balance or equity |
