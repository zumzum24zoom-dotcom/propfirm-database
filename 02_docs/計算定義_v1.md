## このドキュメントの役割

最短合格・最速出金・トータル日数・着金目安の**算出式（手続き）の唯一の正本**。

三本柱の役割分担:
- **用語辞典** = 単語の正本（語句・表記）。算出式は載せない（国語辞典思想）
- **ソース定義v08** = 構成要素の正本（項目・出力形式・判定）。計算は本書を参照する
- **計算定義v1（本書）** = 手続きの正本（算出式）

ソース定義v08・Worker・タイムラインwidget・Page Makerは、計算を本書に従う。各自が独自の計算式を持たない（二重正本の排除）。

---

## 設計思想（マスター設計）

1. **頻度条件と頻度以外の条件を分離する**（次元削減）
2. **条件として存在しないものはinputから除外する**（擬似値0で埋めない。0とは区別する）
3. **初回限定特例を②の枠外に切り出す**（例外処理の独立化）

計算順: ③ → ① → ②
説明順: ① → ② → ③
（実装と説明で順序を分ける）

---

## 共通概念 X

> **X = MAX（対象条件のうち、日数として表現可能なものの達成最速日）**

### 対象条件（Xのinputに含める）
- 最低取引日数
- 一貫性ルール（**Best Day系のみ**。`ceil(100 ÷ X%)` で日数化）
- その他、日数として直接表現可能な制約

### 除外条件（Xのinputに含めない）
- Loss系（日次損失・最大損失）
- 制限系・時間制約系（2分ルール等）
- **最低利益（金額/割合。例: $100・2%・3%）** — 市場条件依存で日数に変換する一般則がないため。出金条件テーブルには記載するが、最速出金スケジュール算出には反映しない
- 公式が条件としていないもの

**原則**: 条件として存在しないものは擬似値0で埋めず、ロジックのinputから除外する。

---

## 最短合格日数

各Stepの `MAX(最低取引日数（合格）, 一貫性達成最速日数)` の合計。

```
最短合格 = Σ（各Stepの X_合格）
X_合格 = MAX(最低取引日数（合格）, 一貫性達成最速日数Best Day系)
```

**詳細**:
- 一貫性はBest Day系(X%)のみ `ceil(100 ÷ X%)` で算出
- 時間制約系・Loss系は除外
- 「条件として存在しない」項目はinputから除外（0で埋めない）
- 全条件が不存在の場合 → フォールバック: `1日 × Step数`
- Instant Funding = `0日`
- 「0日」「制限なし」「即日理論値」は禁止。全条件不存在でも最低 `1日 × Step数`

---

## 最速出金日数

計算順 ③→①→②、説明順 ①→②→③。

### ① X_出金 を算出（頻度以外の出金条件）
```
X_出金 = MAX(最低取引日数（出金）, 一貫性達成最速日数Funded側Best Day系)
```
- 全不存在 → `X_出金 = 0`

### ② 頻度分岐
```
随時 → X_出金
毎週 → MAX(7,  X_出金)
隔週 → MAX(14, X_出金)
月次 → MAX(30, X_出金)
```

### ③ 初回限定特例（公式明示時のみ）
- 公式が初回出金の特例を明示している場合のみ:
```
最速出金(初回) = X_出金
```
- ②の枠外（例外処理）として独立に切り出す

---

## トータル日数

```
トータル日数 = 最短合格 + 最速出金
```

---

## 着金目安

最速出金日数に、出金申請から実際の着金までの処理時間を加算した目安。
```
着金目安 = 最速出金 + 処理時間（公式記載の出金処理日数）
```
処理時間が公式未記載の場合は備考に「記載なし」とし、着金目安は算出しない。

## 検算（Alpha Capital - Alpha One）

設計の妥当性を示す基準ケース。

| 局面 | 計算 | 結果 |
| :--- | :--- | :--- |
| 合格 | MAX(1, なし) = 1日 | 1日 |
| 出金（随時） | X = MAX(40% → ceil(100÷40) = 3) = 3日 → 随時 = 3日 | トータル 1+3 = **4日** |
| 出金（隔週） | X = MAX(5, 3) = 5 → MAX(14, 5) = 14日 | トータル 1+14 = **15日** |

→ マスター正解（4日）と完全一致。

旧ロジックの誤り: 律速A（Min Day Payout）を頻度に関わらず一律採用していたため、Alpha One On-demandを6日と誤算出していた。本v1（旧v2）で頻度分離により正した。
