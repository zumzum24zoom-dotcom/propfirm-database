# Swing High/Low トレンド転換 勝率テストガイド

## 📋 概要

このプロジェクトは、スイングハイロー（値動きの高値と安値のパターン）を使用して、トレンド転換を検出し、その勝率をテストするものです。

### テストするロジック

**上昇トレンドから下降トレンドへの転換パターン:**
```
①. 上昇トレンド中に最高値（Swing High）を形成
②. その後、前回の Swing High より低い高値を形成
③. その過程で前回の安値（Swing Low）を下回る
④. 確定ローソク足でダウントレンド転換シグナル
```

**下降トレンドから上昇トレンドへの転換パターン:**
```
①. 下降トレンド中に最安値（Swing Low）を形成
②. その後、前回の Swing Low より高い安値を形成
③. その過程で前回の高値（Swing High）を上回る
④. 確定ローソク足でアップトレンド転換シグナル
```

## 🔧 インジケーターのセットアップ

### ステップ1: パインスクリプトをコピー

1. TradingView チャートを開く
2. 画面下の「インジケーター」をクリック
3. 「Pine エディターを開く」を選択
4. 新しいスクリプトを作成
5. 以下のファイル内容をコピー＆ペースト:
   - **インジケーター用:** `swing_trend_indicator.pine`
   - **バックテスト用:** `swing_strategy_backtest.pine`

### ステップ2: パラメータ調整

#### インジケーター設定

| パラメータ | 初期値 | 説明 |
|-----------|--------|------|
| **Swing Detection Period** | 5 | スイング判定の期間（バーの本数） |
| **ATR Multiplier** | 1.0 | ATRベースのフィルター倍数（オプション） |

- **Period = 5:** より敏感（シグナルが多い、偽信号も多い）
- **Period = 10:** より確実（シグナル数は減るが精度向上）
- **推奨:** 取引する時間足に応じて5～10の範囲で調整

#### バックテスト設定

| パラメータ | 初期値 | 説明 |
|-----------|--------|------|
| **Swing Detection Period** | 5 | インジケーターと同じ |
| **Risk/Reward Ratio** | 1.5 | リスクリワード比率 |
| **Max Drawdown %** | 20 | 最大ドローダウン制限 |
| **Use Timezone Filter** | true | 東京セッション（9:00-15:00 UTC+9）に限定 |
| **Trade Direction** | BOTH | UP/DOWN/BOTH から選択 |

## 📊 バックテスト結果の見方

### 主要メトリクス

#### 1. 勝率（Win Rate）
```
計算式: (勝ちトレード数 / 総トレード数) × 100%
例: 30勝 / 48トレード = 62.5%
```
- **目標:** 50% 以上（ランダムより良い）
- **推奨:** 55% 以上（期待値プラス）

#### 2. 利益因子（Profit Factor）
```
計算式: 総利益 / 総損失
例: 利益 $3,600 / 損失 $1,270 = 2.84倍
```
- **1.0以下:** システムは赤字
- **1.0～1.5:** 限界的
- **1.5～2.0:** 良好
- **2.0以上:** 優秀

#### 3. 平均獲利 / 平均損失比率
```
計算式: 1トレード当たりの平均利益% / 平均損失%
例: 2.45% / 1.82% = 1.35倍
```
- **良い比率:** 1.5以上
- **リスク/リワード概念を反映

#### 4. 最大ドローダウン（Max Drawdown）
```
計算式: (ピーク時残高 - 最低時残高) / ピーク時残高 × 100%
例: ($12,500 - $10,200) / $12,500 = 18.4%
```
- **管理の必要性:** 20～30% が現実的
- **リスク許容度:** より厳しい設定ほどトレード数は減る

## 🎯 異なる時間足でのテスト方法

### 推奨設定

| 時間足 | Swing Period | パターン | 特徴 |
|-------|-------------|---------|------|
| **15分** | 3～4 | スキャルピング | 多くのシグナル、スプレッド考慮が重要 |
| **1時間** | 5～7 | デイトレード | バランス型（多くのテスターに推奨） |
| **4時間** | 8～10 | スイング | より確実、シグナル数が少ない |
| **日足** | 10～15 | ポジショントレード | 長期、最も信頼性が高い |
| **週足** | 15～20 | 長期投資 | まれなシグナル、高精度 |

### 時間足ごとのテスト手順

1. **チャート選択:** 対象の時間足を表示
2. **パラメータ設定:** 上記の推奨値を入力
3. **バックテスト期間:** 3年以上推奨
4. **統計情報の記録:** パフォーマンステーブルをスクリーンショット

## 💹 結果の分析・改善

### 勝率が低い場合（<50%）

**原因の可能性:**
1. **Period が小さすぎ** → 5→7 に増やす
2. **偽信号が多い** → Timeframe を上げる（1h→4h）
3. **相場が荒い時間帯** → Timezone フィルターを有効化

**改善策:**
```
期間を延ばす → Swing Period を +2～3 増やす
時間足を上げる → 1h → 4h → 1d
条件を厳しくする → 追加フィルター（ボリューム、インジケーター）を追加
```

### 利益因子が低い場合（<1.5）

**原因の可能性:**
1. **損失が大きい** → ストップロス位置が不適切
2. **利益が小さい** → 利確位置が早すぎる
3. **負ける確率が高い** → 市場環境に依存

**改善策:**
```
Risk/Reward Ratio を 2.0 に上げる（利確タイミング延長）
ドローダウン制限を厳しくする（20% → 15%）
シグナル発生後、追加確認ローソク足を待つ（Period + 1本待つ）
```

### ドローダウンが大きい場合（>20%）

**原因の可能性:**
1. **連続損失** → 市場環境が悪い
2. **ポジションサイズが大きい** → リスク設定の見直し
3. **利益確定が遅い** → トレーリングストップの導入

**改善策:**
```
Max Drawdown を 15% に厳しくする
Timezone フィルターで取引時間を限定
ボラティリティベースのポジションサイズ調整
```

## 📈 実装例：パラメータセット

### セット1: 保守的（信頼性重視）
```
Swing Period:      10
Max Drawdown:      15%
Trade Direction:   DOWN のみ（下降転換のみテスト）
Timezone Filter:   ON（東京セッション限定）
```
**特徴:** シグナルは少ないが精度が高い

### セット2: バランス型（汎用）
```
Swing Period:       5～7
Max Drawdown:      20%
Trade Direction:   BOTH
Timezone Filter:   OFF（全時間帯）
```
**特徴:** 推奨設定。多くの相場で機能

### セット3: アグレッシブ（スキャルピング）
```
Swing Period:       3～4
Max Drawdown:      25%
Trade Direction:   BOTH
Timezone Filter:   ON（高流動性時間帯）
```
**特徴:** シグナルが多いが偽信号も増加

## 🔍 外部要因との相互作用

### 注意すべき市場状況

| 状況 | 影響 | 対応 |
|------|------|------|
| **トレンド相場** | ✓ 優秀（最高のシナリオ） | そのまま実行 |
| **レンジ相場** | ✗ 悪い（偽シグナル多）| フィルター追加（MACD、移動平均線確認） |
| **ボラティリティ急増** | × 最悪（損失拡大）| ドローダウン制限を厳しく |
| **経済指標発表時** | × 最悪（スリッページ大） | Timezone で時間帯除外 |

### 相場環境フィルターの追加例

```pinescript
// 移動平均線トレンド確認（オプション）
ma50 = ta.sma(close, 50)
ma200 = ta.sma(close, 200)
isUptrend = ma50 > ma200
isDowntrend = ma50 < ma200

// トレンド確認後のみシグナル
validUpSignal = upSignal and isUptrend
validDownSignal = downSignal and isDowntrend
```

## ⚠️ 重要な注意事項

1. **過去のパフォーマンスは将来の利益を保証しない**
   - マーケット環境の変化により戦略の有効性は低下する可能性がある

2. **バックテスト結果は理想的な環境での数値**
   - スリッページ、スプレッド拡大、執行遅延を完全には反映していない

3. **リアルトレード前に小ロットでテスト**
   - デモ口座 → 0.01ロット → 通常ロット の段階を推奨

4. **市場環境に応じてパラメータを調整**
   - 同じ設定で永遠に利益を得られるわけではない

5. **メンタル面の準備**
   - 連続損失時の心理的対応を事前に計画

## 📞 トラブルシューティング

### Q: パインスクリプトがエラーになる
**A:** Pine Script のバージョン確認。`//@version=5` が正しいバージョンか確認してください。

### Q: シグナルが表示されない
**A:** 
- Period 値が大きすぎる可能性あり
- チャートのズーム/パンで非表示になっていないか確認
- インジケーターの時間足がチャート時間足と合致しているか確認

### Q: バックテスト結果が悪い
**A:**
- テスト期間を拡大（最低3年）
- 異なる Period で複数テスト
- 市場環境の影響（トレンド相場 vs レンジ相場）を確認
- 相場環境フィルター（移動平均線等）の追加を検討
